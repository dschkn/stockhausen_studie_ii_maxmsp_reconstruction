# Max/MSP patch

## Запуск

1. Откройте `StudieII_Generator.maxpat`.
2. Включите DSP кнопкой `ezdac~`.
3. Оставьте небольшие значения `dry gain` и `reverb gain`.
4. Нажмите **ORIGINAL** либо **RANDOM PERMUTATION**.
5. Проверьте, что таблица событий заполнена 380 строками.
6. Нажмите `play`.

Пресет `1.` воспроизводит расчётную скорость. Пресет `3.43` растягивает исходную модель приблизительно до десяти минут. Поскольку случайные seeds дают немного разные общие длительности, десять минут здесь не метрологический акт, а разумная концертная величина.

## Архитектура

### Визуальная часть Max

- `StudieII_Generator.maxpat` — интерфейс и поток данных;
- `StudieII_Voice.maxpat` — пять `cycle~`, суммирование, огибающая, защита от щелчков и панорамирование;
- `poly~ StudieII_Voice 16` — шестнадцать параллельных голосов;
- `jit.cellblock` — десять квадратов и таблица событий;
- `jit.lcd` — партитуроподобное изображение всех сгенерированных прямоугольников;
- `zl.scramble 1 2 3 4 5` — рандомизация исходной перестановки.

### Модули

- `studie2_series_core.js` — чистые операции построения `R1–R5/U1–U5`;
- `studie2_series_engine.js` — связь квадратов с Max;
- `studie2_material_core.js` — частоты, длительности, интенсивности;
- `studie2_material_engine.js` — связь банков с Max;
- `studie2_events_core.js` — процедуры пяти частей и 380 событий;
- `studie2_event_engine.js` — передача событий в таблицу и scheduler;
- `studie2_scheduler.js` — transport и распределение событий между голосами;
- `studie2_score_view.js` — визуализация прямоугольников партитуры;
- `studie2_voice_control.js` — только сообщения для одного голоса; сами осцилляторы находятся в патче.

Таким образом, проект не является одним непрозрачным JavaScript-комом. Логика разделена по уровням, а ключевые музыкальные этапы видны в интерфейсе Max, потому что смотреть на одну коробку `js everything.js` было бы концептуально уныло и педагогически почти преступно.

## Generated data

Сгенерированные таблицы событий не хранятся в репозитории: они создаются заново из выбранного ряда при работе патча. Это уменьшает шум в истории Git, человечество и без того производит достаточно лишних JSON.

## Возможные сообщения Max Console

Если Max не находит `StudieII_Voice.maxpat` или `.js`, значит один из файлов был перемещён отдельно. Верните все файлы в одну папку. Если не создаётся `rev3~`, установлена урезанная конфигурация MSP; сухой тракт можно соединить с `ezdac~` напрямую или оставить wet gain на нуле после восстановления объекта.
